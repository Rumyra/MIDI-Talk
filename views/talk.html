MIDI Specification

Industry experts in 1983

MIDI Manufacturers Association

So it's a technical specification - just like anything else. Made to define a data protocol so digital instruments & equipment could be compatible and talk to each other. Industry experts got together in 1982 and put it together in 1983.

Think of the MMA like the W3C - it's a bunch of industry experts, with an invested interest in making a spec that suits everyones needs. Right. Simples.

It all came about because of the rise of digital instruments - synthesizers, sequencers, digital keyboards etc...

There's lots to it

Core of General MIDI: Messages

Channel Messages & System Messages

Channel messages transmitted on individual channels - depict note on/off, pressure, program change etc...

System messages are system wide - timing signals, start, continue & stop, system reset etc...

the first byte 144 is the message type and channel, 144 is 90 in hex and the 9 says it's a 'Note on' message and the 0 is the channel number. A 'Note on' message has two data bytes, the first is the 'key number' which I'm guessing is the note number, so 60 would be C the second is 'note on velocity' and this has a value of 100, (I'm guessing this means the rate at which the note is turned on)
